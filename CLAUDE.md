# Claude Code Instructions

## Workflow Rules

- **Never auto-commit.** Only create git commits when explicitly asked.
- **Always run tests** after making code changes, and report results before finishing.
- **Ask before installing packages.** Confirm with the user before adding any new dependency.
- **Prefer minimal changes.** Fix or implement only what was asked — do not refactor surrounding code, add comments, or clean up unrelated areas.

## General Preferences

- Keep responses concise and direct.
- No emojis unless explicitly requested.
- Use GitHub-flavored markdown for formatting.
- Prefer editing existing files over creating new ones.
- Do not create documentation files unless explicitly requested.
