How it works — end to end
📋
1
Upload
Weekly member spreadsheet uploaded. Active members extracted, issues flagged (cancellations, holds, allergens).
🗺️
2
Assign to Routes
Each member's ZIP code is matched to a borough and delivery day. Overlapping ZIPs load-balance across routes.
📍
3
Geocode
Each address is converted to GPS coordinates. Results cached — addresses seen before skip this step.
⚡
4
Optimize Order
Stops are sequenced from Tribeca depot using nearest-neighbour + 2-opt improvement to minimise total mileage.
📄
5
Generate Outputs
Driver manifest, interactive map, and kitchen packing list exported per route.

📏 Route Rules
  • Max 25 stops per route (28 if stops are tightly clustered)
  • Max 25 miles per route including depot legs
  • ZIP codes in two routes: assigned to the route with fewer stops (load-balancing)
  • Ridgewood (11385) always assigned to Brooklyn Route B — better parking access near depot
  • Members with allergens, cancel/hold notes, or failed geocodes are flagged for review
⚡ Performance
  • Addresses cached after first geocode — repeat weeks are near-instant
  • Parallel geocoding: up to 10 addresses processed simultaneously
  • Typical run time: < 30 seconds for a fully-cached spreadsheet
  • New addresses add ~2–3 seconds each
